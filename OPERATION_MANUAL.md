# 中文地址语义匹配系统 - 操作手册

## 一、系统概述

本系统基于阿里达摩院 MGeo 地址语义匹配模型，实现企业地址与标准地址库的高效精准匹配。系统采用"粗召回+精排"两阶段架构，支持千万级数据量处理。同时提供独立的MGeo地址相似度匹配功能。

### 匹配流程
1. **粗召回阶段**: 使用 MGeo backbone 模型将地址向量化，通过 pgvector 检索每个企业最相似的候选地址，同时按相似度阈值过滤低分候选
2. **精排阶段**: 使用 MGeo 精排模型对候选地址进行两两比较，输出三个概率值（exact_match/partial_match/not_match），按匹配得分确定最终匹配结果，同时按相似度阈值二次过滤

### 精排候选排序逻辑
1. **主排序标准**: `exact_match` - 按精确匹配概率最高值筛选最佳匹配候选
2. **次排序标准**: `not_match` - 当 exact_match 相同时，优先选择不匹配概率更低的候选

### 相似度阈值过滤
相似度阈值在两个阶段均生效，实现双重过滤：
- **粗召回阶段**：SQL `WHERE` 条件过滤，在数据库层面排除向量相似度低于阈值的候选
- **精排阶段**：Python 代码中再次过滤，确保即使粗召回未过滤，精排阶段仍可按阈值过滤

> 阈值设为 0 表示不过滤，保留所有候选；阈值越高，过滤越严格。

### 匹配状态说明
| 状态 | 含义 | 判断条件 |
|------|------|---------|
| 精确匹配 | 企业地址与标准地址高度一致 | exact_match 为三个概率中最大值 |
| 部分匹配 | 企业地址与标准地址部分一致 | partial_match 为三个概率中最大值 |
| 不匹配 | 无候选地址或not_match概率最高 | not_match 为三个概率中最大值或无候选 |

### 运行模式
系统自动检测电脑配置：
- **有独立显卡（GPU）**: 自动使用GPU加速运行，界面显示"🖥️ GPU模式运行"
- **无独立显卡**: 使用CPU运行，界面显示"💻 CPU模式运行"

系统采用 Streamlit 侧边栏导航布局：

功能菜单：
- ⚙️ 数据库配置
- 📊 向量预处理
- 🔍 地址匹配
- 📋 结果管理
- 📝 系统日志

## 三、操作流程

### 3.1 数据库配置

1. 在侧边栏点击【⚙️ 数据库配置】
2. 填写数据库连接信息：
   - **数据库主机**: 数据库服务器地址（默认 localhost）
   - **端口**: PostgreSQL 端口（默认 5432）
   - **数据库名**: 数据库名称（默认 postgres）
   - **模式（Schema）**: 数据库模式名（默认 public，可改为自定义模式如 ai）
   - **用户名**: 数据库用户名（默认 postgres）
   - **密码**: 数据库密码
3. 点击【测试连接】验证配置是否正确

> **重要**: 系统连接数据库后会自动设置 `search_path` 到用户指定的 Schema，
> 所有表操作（创建向量表、召回结果表等）都会在该 Schema 下执行。
> 支持中文表名，系统会自动对SQL标识符进行双引号引用。

### 3.2 向量预处理

> 首次匹配前必须完成此步骤

1. 在侧边栏点击【📊 向量预处理】
2. 分别配置企业表和标准地址表的字段映射：
   - **选择表**: 从下拉框选择企业表/标准地址表
   - **字段选择**: 选择表后，所有字段下拉框自动默认选中第一个字段
   - 如需修改，可手动切换到其他字段
3. 设置处理参数（批次大小等）
4. 点击【启动向量化】开始处理
5. 等待处理完成，系统会自动创建向量索引
6. 向量化过程中会显示当前运行模式（GPU/CPU）

> **提示**: 更换选择的表后，字段选择会自动重置为第一个字段，避免字段映射错误。

### 3.3 地址匹配

1. 在侧边栏点击【🔍 地址匹配】
2. 页面顶部显示当前运行模式（GPU/CPU）
3. 系统支持分步执行：
   - **数据粗召回**: 选择向量表，配置参数（包括相似度阈值）后点击【启动数据粗召回匹配】
   - **MGeo精确匹配**: 粗召回完成后，点击【继续MGeo精确匹配】或【启动MGeo精确匹配】

#### 粗召回参数说明
| 参数 | 说明 | 建议值 |
|------|------|--------|
| 相似度阈值 | 向量相似度低于此值的候选将被过滤 | 0.5-0.8（0表示不过滤） |
| 召回候选数 | 每个企业召回的候选数量 | 10-50 |

#### 粗召回完成后操作
- 📊 **数据查看**: 直接跳转至"结果管理"的"粗召回数据"页签
- ▶️ **继续MGeo精确匹配**: 执行MGeo精排
- ↩️ **返回地址匹配**: 返回初始匹配页面
- 🔄 **重新进行粗召回匹配**: 清空召回结果重新匹配

#### 精排完成后操作
- 📊 **查看匹配结果**: 直接跳转至"结果管理"的"精排匹配结果"页签
- 🔄 **重新开始完整匹配**: 重置所有状态重新匹配
- ↩️ **返回地址匹配**: 返回初始匹配页面

### 3.4 MGeo地址相似度匹配

独立的地址相似度匹配功能，直接调用MGeo模型对地址A和地址B进行逐行匹配，无需向量召回。

#### 文件输入模式
1. 选择数据输入方式为"上传文件"
2. 上传CSV或Excel文件
3. 选择地址字段A和地址字段B
4. 点击【启动相似度匹配】
5. 匹配完成后：
   - 📥 **下载Excel格式**: 将匹配结果下载为Excel文件
   - 📥 **下载CSV格式**: 将匹配结果下载为CSV文件
   - 🔄 **重新匹配**: 重置状态重新匹配
   - ↩️ **返回地址匹配**: 返回匹配页面

> 文件输入模式下，匹配结果不会写入数据库，仅提供下载功能。

#### 数据库表输入模式
1. 选择数据输入方式为"数据库表"
2. 选择数据表和地址字段
3. 点击【启动相似度匹配】
4. 匹配完成后：
   - 📊 **结果查看**: 直接跳转至"结果管理"的"MGeo地址相似度匹配结果"页签
   - 🔄 **重新匹配**: 重置状态重新匹配
   - ↩️ **返回地址匹配**: 返回匹配页面

> 数据库表输入模式下，匹配结果会写入mgeo_similarity_results表，并自动生成_mgeo副本表。

### 3.5 结果管理

1. 在侧边栏点击【📋 结果管理】
2. 通过页签选择器切换查看不同结果：
   - **粗召回数据**: 分页浏览、导出CSV/Excel
   - **精排匹配结果**: 筛选、分页浏览、行选择、人工纠正、导出、统计分析
   - **MGeo地址相似度匹配结果**: 筛选、分页浏览、导出、统计分析、副本表查看

> 从地址匹配页面点击"数据查看"/"查看匹配结果"/"结果查看"按钮，会自动跳转到对应页签。

#### 精排匹配结果筛选功能
- 按匹配状态筛选（精确匹配/部分匹配/不匹配）
- 按精确匹配概率范围筛选
- 按部分匹配概率范围筛选
- 按不匹配概率范围筛选
- 关键词搜索（企业名/企业地址/标准地址）

#### 人工纠正功能

对精排匹配结果不满意时，可通过人工纠正功能从粗召回数据中选择更合适的匹配结果。

**操作步骤**：
1. 在精排匹配结果页面，用鼠标点击行数据选中一条或多条记录（支持多选）
2. 选中后出现"🔧 人工纠正"按钮
3. 点击"🔧 人工纠正"，系统自动跳转至粗召回数据页面，并按企业ID筛选出相关数据
4. 页面顶部显示黄色提示"请选择需要更改的数据"
5. 在数据列表中勾选需要标记的行（支持多行勾选）
6. 点击"✅ 标记为精确匹配"按钮
7. 在确认对话框中点击"✔️ 确认提交"
8. 系统自动跳回精排匹配结果页面，选中的企业匹配数据已更新为标记的数据

> **说明**：人工纠正后，对应记录的匹配状态变为"精确匹配"，精确匹配概率设为1.0，纠正来源标记为"人工纠正"。
> 可随时点击"↩️ 返回"按钮退出人工纠正模式。

#### 导出功能
- 导出为 CSV 格式
- 导出为 Excel 格式（大数据量自动分工作表）

#### 匹配统计
- 总记录数、精确匹配数、部分匹配数、不匹配数、人工纠正数
- 各状态占比
- 三类概率均值分布
- 匹配状态分布饼图
- 概率均值柱状图
- 匹配来源分布饼图（自动匹配 vs 人工纠正）

### 3.6 系统日志

1. 在侧边栏点击【📝 系统日志】
2. 查看系统运行日志
3. 支持按日志级别筛选
4. 可运行向量调试测试

## 四、参数配置建议

### 4.1 相似度阈值
- **不过滤模式**: 阈值设为 0，保留所有候选
- **宽松过滤**: 阈值 0.3-0.5，过滤明显不匹配的候选
- **适中过滤**: 阈值 0.5-0.7，平衡召回率和精度
- **严格过滤**: 阈值 0.7-0.9，只保留高相似度候选

### 4.2 粗召回候选数
- **高精度模式**: 候选数 10-20，减少精排计算量
- **平衡模式**: 候选数 20-50，兼顾精度和召回率
- **高召回模式**: 候选数 50-100，最大化召回率

### 4.3 性能优化
- **GPU 环境**: 模型推理批次可设为 32-64
- **CPU 环境**: 模型推理批次建议设为 8-16
- **内存受限**: 减小数据库读取批次（如 500）

## 六、常见场景操作

### 场景1: 首次全量匹配
1. 配置数据库连接（填写正确的Schema名）
2. 执行向量预处理（企业表 + 标准地址表）
3. 配置匹配参数（设置相似度阈值），启动粗召回匹配
4. 粗召回完成后点击【继续MGeo精确匹配】
5. 精排完成后点击【查看匹配结果】跳转至结果管理

### 场景2: 分步执行（先粗召回后精排）
1. 在地址匹配页面点击【启动数据粗召回匹配】
2. 等待粗召回完成
3. 点击【数据查看】跳转至结果管理查看粗召回数据
4. 返回地址匹配页面，点击【启动MGeo精确匹配】
5. 等待精排完成，点击【查看匹配结果】

### 场景3: 使用非public模式（Schema）
1. 在数据库配置页面填写模式名（如 ai）
2. 确保该Schema已在数据库中创建（`CREATE SCHEMA ai;`）
3. 选择该Schema下的企业表和标准地址表（支持中文表名）
4. 系统会自动在该Schema下创建向量表和结果表

### 场景4: MGeo相似度匹配（文件输入）
1. 在地址匹配页面找到"MGeo地址相似度匹配"区域
2. 选择"上传文件"模式
3. 上传包含地址A和地址B的CSV/Excel文件
4. 选择对应的地址字段
5. 点击【启动相似度匹配】
6. 匹配完成后点击"下载Excel格式"或"下载CSV格式"保存结果

### 场景5: MGeo相似度匹配（数据库表输入）
1. 在地址匹配页面找到"MGeo地址相似度匹配"区域
2. 选择"数据库表"模式
3. 选择数据表和地址字段
4. 点击【启动相似度匹配】
5. 匹配完成后点击【结果查看】跳转至结果管理

### 场景6: 筛选导出精确匹配结果
1. 在结果管理页面，选择"精排匹配结果"页签
2. 匹配状态筛选选择"精确匹配"
3. 设置概率范围筛选条件
4. 点击导出按钮选择格式

### 场景7: 人工纠正匹配结果
1. 在结果管理页面，选择"精排匹配结果"页签
2. 找到匹配不正确的记录，用鼠标点击行选中（支持多选）
3. 点击"🔧 人工纠正"按钮
4. 系统跳转至粗召回数据页面，显示该企业的所有候选地址
5. 勾选正确的候选地址行
6. 点击"✅ 标记为精确匹配"
7. 在确认对话框中点击"✔️ 确认提交"
8. 系统自动跳回精排匹配结果页面，数据已更新

## 七、模型部署说明

### 模型文件部署方式

系统支持多种模型部署方式，按优先级自动搜索：

1. **项目目录部署（推荐）**：将模型放置在 `address_match/models/iic/` 目录下
2. **ModelScope缓存目录**：模型自动下载到 `~/.cache/modelscope/hub/models/iic/`
3. **环境变量指定**：通过 `MODELSCOPE_CACHE` 环境变量指定缓存目录
4. **在线下载**：网络可用时自动从 ModelScope/HuggingFace 下载

> 部署到新电脑时，推荐使用方式1，将模型文件随项目一起复制。
