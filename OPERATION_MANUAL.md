# 中文地址语义匹配系统 - 使用说明

## 一、系统概述

本系统是一个基于深度学习和向量检索的中文地址语义匹配平台，主要功能包括：

- **向量预处理**：将企业地址和标准地址转换为向量并存储到数据库
- **地址匹配**：支持库表输入和文件输入两种模式，实现企业地址与标准地址的智能匹配
- **MGeo地址相似度**：直接比较两个地址文本的相似度
- **结果管理**：查看、筛选、导出匹配结果，支持人工纠正
- **系统日志**：查看实时日志和数据库存储日志，支持向量调试测试

## 二、界面导航

系统左侧导航栏包含五个主要功能模块：

1. **数据库配置** - 连接PostgreSQL数据库
2. **向量预处理** - 生成和存储地址向量
3. **地址匹配** - 执行地址匹配任务
4. **结果管理** - 查看和管理匹配结果
5. **系统日志** - 查看日志和运行调试测试

## 三、功能详解

### 3.1 数据库配置

首次使用系统前，需要先配置数据库连接。

**操作步骤**：
1. 在左侧导航栏点击"数据库配置"
2. 填写数据库连接信息：
   - **数据库主机**: PostgreSQL服务器地址（默认 localhost）
   - **端口**: 数据库端口（默认 5432）
   - **数据库名**: 数据库名称（默认 postgres）
   - **模式（Schema）**: 数据库模式名（默认 public，可自定义如 ai）
   - **用户名**: 数据库用户名（默认 postgres）
   - **密码**: 数据库密码
3. 点击"连接数据库"按钮
4. 连接成功后，系统会自动检测并创建所需的数据表

**注意事项**：
- 连接成功后，系统会自动设置 `search_path` 到用户指定的 Schema，所有表操作在该 Schema 下执行
- 支持中文表名，系统会自动对SQL标识符进行双引号引用
- 连接成功后左侧导航栏会显示"数据库已连接"状态

### 3.2 向量预处理

向量预处理是将企业地址和标准地址转换为向量并存储到数据库的过程，是后续匹配的基础。

**操作步骤**：
1. 在左侧导航栏点击"向量预处理"
2. 选择向量表类型：
   - **企业向量表**: 存储企业地址的向量
   - **标准地址向量表**: 存储标准地址的向量
3. 配置数据源：
   - **库表输入**: 从数据库表读取地址数据
     - 选择包含地址的数据库表
     - 选择地址列名
     - 选择ID列名（用于标识地址）
     - 选择企业名称列（可选）
   - **文件输入**: 从CSV/Excel文件读取地址数据
     - 上传CSV或Excel文件
     - 选择地址列名
     - 选择ID列名
     - 选择企业名称列（可选）
4. 配置向量生成参数：
   - **批处理大小**: 每次处理的地址数量（默认1000，根据内存调整）
   - **设备**: CPU或GPU（系统会自动检测GPU可用性）
5. 点击"开始生成向量"按钮

**注意事项**：
- 向量预处理是匹配的前提，必须先完成向量生成才能进行匹配
- 如果数据量较大，建议分批处理或调整批处理大小
- 系统会自动跳过已存在的地址（根据ID判断），支持增量更新
- 向量表支持中文表名

### 3.3 地址匹配

地址匹配是系统的核心功能，支持两种输入模式。

#### 3.3.1 库表输入模式

从数据库表读取企业地址，与标准地址向量表进行匹配。

**操作步骤**：
1. 在左侧导航栏点击"地址匹配"
2. 选择输入模式为"库表输入"
3. 选择或创建标签（用于隔离不同批次的数据）
4. 配置数据源：
   - 选择企业地址表
   - 选择企业ID列
   - 选择企业名称列
   - 选择企业地址列
5. 配置匹配参数：
   - **Top-K 召回数**: 粗召回阶段返回的候选数量（默认10）
   - **相似度阈值**: 向量相似度过滤阈值（默认0，不过滤）
   - **批处理大小**: 每次处理的企业数量（默认500）
6. 点击"开始匹配"按钮

**匹配流程**：
1. 从企业地址表读取数据
2. 粗召回：使用向量相似度从标准地址向量表中召回Top-K候选
3. 精排：使用MGeo模型对候选进行精确匹配、部分匹配、不匹配分类
4. 结果存储到数据库

#### 3.3.2 文件输入模式

从CSV/Excel文件读取企业地址，进行匹配。

**操作步骤**：
1. 选择输入模式为"文件输入"
2. 上传CSV或Excel文件
3. 选择列映射（企业ID、企业名称、企业地址）
4. 配置匹配参数（同库表输入）
5. 点击"开始匹配"按钮

**注意事项**：
- 文件输入模式下，匹配结果不会写入数据库，而是以文件形式下载
- 支持CSV和Excel格式导出

#### 3.3.3 MGeo地址相似度匹配

直接比较两个地址文本的相似度，无需向量预处理。

**操作步骤**：
1. 在"地址匹配"页面选择"MGeo地址相似度匹配"
2. 选择输入模式：
   - **文件输入**: 上传包含两列地址的CSV/Excel文件
   - **库表输入**: 从数据库表读取两列地址
3. 配置列映射（地址A列、地址B列）
4. 配置批处理大小
5. 点击"开始MGeo相似度匹配"按钮

**注意事项**：
- MGeo相似度匹配结果会存储到 `mgeo_similarity_results` 表（库表输入模式）
- 文件输入模式下结果以下载方式获取

### 3.4 结果管理

查看、筛选、导出匹配结果，支持人工纠正。

#### 3.4.1 粗召回数据

查看粗召回阶段的候选数据。

**功能**：
- 按关键词搜索
- 分页浏览（支持首页、上一页、下一页、末页、页码跳转）
- 导出为CSV或Excel格式

#### 3.4.2 精排匹配结果

查看精排后的匹配结果。

**筛选条件**：
- 匹配状态（精确匹配/部分匹配/不匹配）
- 精确匹配概率范围
- 部分匹配概率范围
- 不匹配概率范围
- 关键词搜索

**功能**：
- 分页浏览
- 多行选择（勾选多条记录）
- **人工纠正**: 选中记录后点击"人工纠正"按钮，可修改匹配结果
- 导出为CSV或Excel格式

**人工纠正操作**：
1. 在精排匹配结果表格中勾选需要纠正的记录
2. 点击"人工纠正"按钮
3. 在纠正页面选择正确的标准地址
4. 保存纠正结果

#### 3.4.3 MGeo地址相似度匹配结果

查看MGeo相似度匹配的结果。

**功能**：
- 按匹配状态和概率范围筛选
- 分页浏览
- 导出为CSV或Excel格式
- 查看MGeo副本表数据

#### 3.4.4 匹配统计

系统会自动统计匹配结果并展示图表：
- 匹配状态分布（饼图）
- 三类匹配概率均值分布（柱状图）
- 匹配来源分布（自动匹配 vs 人工纠正）

### 3.5 系统日志

查看系统运行日志和调试信息。

#### 3.5.1 实时日志（内存）
- 显示内存中的最近500条日志
- 支持清空内存日志

#### 3.5.2 数据库存储日志
- 显示持久化存储在数据库中的日志
- 支持按日志级别筛选（全部/WARNING/ERROR）
- 支持清空数据库日志

#### 3.5.3 向量调试测试
- 测试Python端向量化
- 检查数据库表结构
- 检查数据库向量数据
- 测试向量插入

## 四、标签功能

标签用于隔离不同批次的匹配数据，每个标签有独立的召回表和匹配表。

**使用步骤**：
1. 在【地址匹配】页面选择或创建标签
2. 输入标签名称（如"福田区"）
3. 系统会自动生成表名前缀（拼音/英文）
4. 执行匹配时数据会存储到该标签关联的表中
5. 在【结果管理】页面可按标签筛选查看对应数据
6. 删除标签时会同时删除关联的数据表

## 五、匹配逻辑说明

### 5.1 粗召回

使用向量相似度从标准地址中召回候选：
- 计算企业地址向量与标准地址向量的余弦相似度
- 返回相似度最高的Top-K个候选
- 可通过相似度阈值过滤低相似度候选

### 5.2 精排

使用MGeo模型对候选进行分类：
- 输入：企业地址 + 标准地址
- 输出：精确匹配概率、部分匹配概率、不匹配概率
- 候选排序：
  1. 按精确匹配概率降序
  2. 精确匹配概率相同时，按不匹配概率升序
- 匹配状态：取三个概率中的最大值

### 5.3 相似度阈值

阈值在两个阶段均生效：
- **粗召回阶段**：SQL WHERE条件过滤，数据库层面排除低相似度候选
- **精排阶段**：Python代码过滤，确保精排结果符合阈值要求

## 七、常见问题与解决

### Q1: 连接数据库失败
- 检查PostgreSQL服务是否启动
- 检查主机地址、端口、用户名、密码是否正确
- 检查防火墙是否允许连接
- 检查pgvector扩展是否已安装

### Q2: 向量预处理速度慢
- 使用GPU模式（如有NVIDIA显卡）
- 增大批处理大小（根据内存调整）
- 确保模型已下载到本地，避免重复下载

### Q3: 匹配结果不准确
- 检查向量预处理是否完成
- 调整Top-K召回数（增大可获得更多候选）
- 调整相似度阈值（根据数据特点设置合适值）
- 使用人工纠正功能修正错误结果

### Q4: 内存不足
- 减小批处理大小
- 使用CPU模式运行
- 关闭其他占用内存的程序

### Q5: 导出Excel失败
- 安装openpyxl库：`pip install openpyxl`
- 检查磁盘空间是否充足

### Q6: 页面显示异常
- 刷新浏览器页面
- 清除浏览器缓存
- 检查Streamlit版本是否兼容

### Q7: 翻页功能报错
- 确保使用最新版本代码
- 已使用on_click回调机制解决widget绑定问题
- 如仍报错，尝试清除浏览器缓存后重新访问

## 八、最佳实践

1. **数据准备**：确保地址数据清洗完整，去除多余空格和特殊字符
2. **向量预处理**：先完成标准地址向量生成，再生成企业地址向量
3. **匹配参数**：根据数据规模调整批处理大小，避免内存溢出
4. **结果检查**：定期查看匹配统计，发现异常及时调整参数
5. **人工纠正**：对关键数据进行人工复核，提高匹配准确率
6. **标签管理**：使用标签隔离不同批次的数据，便于管理和追溯
